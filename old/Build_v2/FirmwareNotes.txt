Board/package used for flashing:
esp32:esp32:nodemcu-32s

Purpose:
Fix MPU startup state where MPU1 gyro was live but MPU1 acceleration was
capturing as all zeros.

Exact firmware changes:

1. Added MPU register definitions:

    #define REG_SIGNAL_RESET 0x68
    #define REG_USER_CTRL    0x6A
    #define REG_PWR_MGMT_2   0x6C

2. Replaced the old MPU setup start:

    writeMPU(addr, REG_PWR_MGMT_1, 0x01);
    delay(100);

   with a full reset and signal-path reset:

    writeMPU(addr, REG_PWR_MGMT_1, 0x80);
    delay(100);
    writeMPU(addr, REG_SIGNAL_RESET, 0x07);
    delay(20);
    writeMPU(addr, REG_PWR_MGMT_1, 0x01);
    writeMPU(addr, REG_PWR_MGMT_2, 0x00);
    writeMPU(addr, REG_USER_CTRL, 0x00);

3. Kept the original MPU runtime configuration:

    writeMPU(addr, REG_CONFIG, 0x03);
    writeMPU(addr, REG_SMPLRT_DIV, 0x00);
    writeMPU(addr, REG_GYRO_CONFIG, 0x00);
    writeMPU(addr, REG_ACCEL_CONFIG, 0x00);
    writeMPU(addr, REG_INT_ENABLE, 0x01);

4. Added a settle delay at the end of setupMPU():

    delay(100);

5. Kept high-speed serial unchanged:

    #define STREAM_BAUD 1000000

6. Kept I2C at the original high-speed setting:

    Wire.setClock(1000000);

What the new setup sequence does:

- REG_PWR_MGMT_1 = 0x80 performs a full MPU device reset.
- REG_SIGNAL_RESET = 0x07 resets accel, gyro, and temperature signal paths.
- REG_PWR_MGMT_1 = 0x01 wakes the MPU and selects the gyro X clock source.
- REG_PWR_MGMT_2 = 0x00 ensures no accel or gyro axes are in standby.
- REG_USER_CTRL = 0x00 ensures auxiliary/FIFO/I2C master features are off.
- The final delay gives the sensor time to settle before streaming starts.

Command-line flashing:

    cd Build_v2/MCU
    make flash FLASH_PORT=/dev/cu.usbserial-0001

Command-line monitor:

    cd Build_v2/MCU
    make monitor MONITOR_PORT=/dev/cu.usbserial-0001

Arduino IDE flashing:

1. Install "ESP32 by Espressif Systems" from Boards Manager.
2. Select board "NodeMCU-32S".
3. Select the correct serial port.
4. Open Build_v2/MCU/firmware_1.1.ino.
5. Upload normally.

Serial stream expected by Python:

    baudRate: 1000000
    recordFormat: <I12hf
    bufferSampleCount: 32

The Python capture/live code expects this exact packet format.
