# Fan Vibration Build

This repo is now a stripped fan-vibration pipeline built around one
`MPU6050` and two live stream modes:

- `all_1k`
- `gyro_8k`

The Python side is aligned to the repo rules:

- direct top-down scripts
- one shared buffer path
- one shared signal path
- one shared ML path
- one shared charting path

The repo now keeps only the active fan-vibration pipeline.
Old example code, legacy build output, caches, and generated chart/build
artifacts have been removed from the working tree.

## Active Files

- `src/buffer.py`
  framed serial transport, mode config, CSV row decode
- `src/signals.py`
  raw time axes, raw FFT axes, visual magnitude, summed FFT
- `src/ml.py`
  direct time + FFT stream model input and PyTorch model
- `src/rules.py`
  light band rules
- `src/charting.py`
  time grid, FFT grid, live summary plot
- `src/capture.py`
  capture one live mode into CSV
- `src/analysis.py`
  read one CSV and save charts
- `src/training.py`
  train one mode-specific model
- `src/live.py`
  live framed stream + ML inference
- `src/visualiser.py`
  live magnitude + summed FFT view
- `MCU/firmware_0.1.ino`
  ESP32-H2 firmware with two stream modes

Generated output:

- `MCU/build/`
  compile output and staged Arduino sketch, regenerated by `make -C MCU compile`
- `outputs/charts/`
  chart output, regenerated by analysis runs

These generated paths are ignored by `.gitignore`.

## Stream Modes

### `all_1k`

- source data:
  `ax ay az gx gy gz`
- target sample rate:
  `1000 Hz`
- payload sample:
  `"<6h"`
- frame samples:
  `128`

### `gyro_8k`

- source data:
  `gx gy gz`
- target sample rate:
  `8000 Hz`
- payload sample:
  `"<3h"`
- frame samples:
  `128`

`gyro_8k` is gyro-only on purpose.
This keeps the high-rate mode focused on the fastest possible stream.

## Framed Binary Protocol

The MCU no longer sends loose raw packets.
It now sends framed batches.

Frame header layout:

- `magic`
  `0xAABBCCDD`
- `version`
  `1`
- `mode`
  `1 = all_1k`
  `2 = gyro_8k`
- `sampleCount`
- `sequence`
- `startSample`
- `sampleRateHz`
- `payloadBytes`

Host command flow:

- send `STOP\n`
- send `MODE ALL_1K\n` or `MODE GYRO_8K\n`
- send `START\n`

The firmware accepts these commands even after streaming has already started.

The current host flow is:

- open the serial link
- optionally read the ready banner if it appears
- send `STOP\n`
- send `MODE ...\n`
- send `START\n`

The host does not rely on toggling `DTR` or forcing a USB reboot.
That keeps the startup path stable on macOS USB CDC ports.

## Signal Derivation

Raw ML signals:

- all live time axes for the selected mode
- all raw FFT axes for the selected mode

Visual-only derived signals:

- vector magnitude from the visual axes
- summed FFT from the visual axes
- fundamental from the summed FFT
- BPFO and BPFI bands from the summed FFT

Current visual axes:

- `all_1k`
  gyro `gx gy gz`
- `gyro_8k`
  gyro `gx gy gz`

Saved charts:

- raw time grid
- raw FFT grid
- summary chart with magnitude and summed FFT

Live visualiser:

- time plot uses sample index on the x axis
- time magnitude is fixed to `0..500`
- FFT x range is fixed by mode
- FFT y range auto-scales each refresh

## Hardware Result

Measured on the current `ESP32-H2 + MPU6050 + I2C` path in this repo with
MPU FIFO reads and framed serial batches:

- `all_1k`
  about `999.7 Hz`
- `gyro_8k`
  about `7998.7 Hz`

Current repo conclusion:

- `all_1k` is stable at the target rate
- `gyro_8k` is stable at the target rate
- the FIFO path is the key change that made true `8 kHz` gyro capture work

The current high-rate path is:

- MPU gyro configured for `8 kHz`
- gyro-only FIFO feed
- FIFO burst reads on the MCU
- framed binary batches over USB CDC

## Commands

Install dependencies:

```bash
python3 -m pip install numpy pandas pyserial matplotlib torch
```

Compile MCU firmware:

```bash
make -C MCU compile
```

Flash MCU firmware:

```bash
make -C MCU flash
```

Capture `all_1k`:

```bash
python3 src/capture.py all_test.csv all_1k
```

Capture `gyro_8k`:

```bash
python3 src/capture.py gyro_test.csv gyro_8k
```

Analyse one capture:

```bash
python3 src/analysis.py data/capture/all_test.csv
```

Train one mode:

```bash
python3 src/training.py all_1k
```

Run live ML:

```bash
python3 src/live.py all_1k
```

Run live visualiser:

```bash
python3 src/visualiser.py gyro_8k
```

Run smoke test:

```bash
python3 src/offline_smoke.py
```
